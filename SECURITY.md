# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| latest 0.x | ✅ |

Both `typeorm-entity-history` and `nestjs-typeorm-history` follow this policy.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use one of these private channels:

- **GitHub private vulnerability reporting** (preferred): [Report a vulnerability](https://github.com/shinxz12/entity-history/security/advisories/new)
- **Email**: ngocbthe@gmail.com

Include as much of the following as you can: affected package and version, a description of the issue and its impact, and steps or a proof-of-concept to reproduce it.

You can expect an acknowledgment within a few days. Once the issue is confirmed and a fix is released, the vulnerability will be disclosed via a GitHub Security Advisory with credit to the reporter (unless you prefer to stay anonymous).
